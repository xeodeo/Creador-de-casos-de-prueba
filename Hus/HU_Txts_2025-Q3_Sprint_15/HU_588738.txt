Landing validación fecha de expedición cedula - OTP Paso 2.

Criterios de aceptacion (ID: Custom.6228a8da-4910-4f16-af9d-1f0a5103b7d2):
----------------
Si el servicio web (GetEstadoValidacionFechaExp) retorna que la línea está en estado Pendiente_Validacion_Cliente o Bloqueada, se debe de enviar un código OTP a la línea móvil éxito ingresada, usando el servicio web para enviar OTPs. Usar el servicio web de la API Autogestión. (Configuración Línea Favorita) --> https://wolframio.grupo-exito.com/apigw/v1/ZZMQMWVXSQRYCHSV/api/SubscriptionBalance/EnviarOTP  Luego se debe activar lo siguiente :  Una opción para que el cliente ingrese el código OTP y ser validado.  un botón para reenviar el OTP en caso de ser necesario. Un botón para confirmar el código OTP Mostrar un texto con la siguiente información : Hemos enviado un Código de Seguridad a la Línea indicada, para continuar por favor ingrésalo acá.   Cuando el cliente ingrese el código OTP y le de en validar se debe consumir el servicio web para confirmar si el código OTP es correcto. Usar el servicio web de la API Autogestión. (Configuración Línea Favorita). --> https://wolframio.grupo-exito.com/apigw/v1/ZZMQMWVXSQRYCHSV/api/SubscriptionBalance/ValidarOTP Si el código OTP ingresado por el cliente no es correcto, se debe mostrar un mensaje indicando que la información no es correcta, por favor validar. Si el código OTP ingresado por el cliente es correcto, entonces se debe de consumir un servicio web para validar si la fecha de expedición es correcta. Cointunar en el PASO 3.     Cuando se consuma el servicio web de Móvil Éxito (GetEstadoValidacionFechaExp) para validar y la respuesta sea diferente Pendiente_Validacion_Cliente o Bloqueada, entonces mostrar un mensaje que sea: "Los datos ingresados no tienen pendientes de validación" --> Mensaje parametrizable
